一、项目架构概览
--------------------------------------------------------------------------------

功能路径:
  SD卡(TF) → sdcmd_ctrl → sd_reader → sd_file_reader → img_ram → vga_ctrl → VGA

模块分工:
  sdcmd_ctrl     : SD CMD 线命令/响应收发，CRC7 检查，48-bit 帧协议
  sd_reader      : SD 卡初始化状态机 (CMD0→CMD8→CMD55→ACMD41→CMD2→CMD3→CMD7→CMD16)
                   + CMD17 单扇区读取 + DAT0 数据接收
  sd_file_reader : FAT32 解析 (MBR→Boot扇区→根目录→文件数据)，查找 IMAGE.BIN
                   写入 img_ram (76800 字节，320×240)
  img_ram        : Cyclone V M10K 双端口 BRAM，8-bit 数据宽度
  vga_ctrl       : 640×480@60Hz VGA 时序，320×240 → 640×480 2x 放大，RGB332→RGB888

SD 协议模式: SD 原生模式 (非SPI)，1-bit 数据线
时钟策略:   初始化 <400kHz (SLOWCLKDIV=256)，传输后切换快速时钟 (FASTCLKDIV=4)

--------------------------------------------------------------------------------
二、验证分层架构

  Layer 3: 系统级 (top_sd_vga)       — FAT32 端到端功能验证

四、SD 卡行为模型 (sd_card_model.sv) — 验证核心基础
--------------------------------------------------------------------------------

实现方式: 独立 SystemVerilog module，直连 sdclk / sdcmd(tri) / sddat[3:0](tri)

命令解析: sdclk 上升沿自动采样 CMD 线 48-bit 帧，解析 cmd_index + arg

响应分发表:
  CMD  |  响应类型          |  关键行为
  -----|---------------------|--------------------------------------------------
  CMD0 |  无响应             |  进入 idle 状态
  CMD8 |  R7 (48-bit)        |  echo check pattern (0x1AA)
  CMD55|  R1 (48-bit)        |  设置 APP_CMD 标志
  ACMD41| R3 (48-bit, 无CRC)|  可配置 busy 循环次数后置 bit31=1 (power_up_status)
  CMD2 |  R2 (136-bit)       |  返回假 CID
  CMD3 |  R6 (48-bit)        |  返回可配置 RCA
  CMD7 |  R1b (48-bit+busy)  |  选中卡
  CMD16|  R1 (48-bit)        |  确认 block length
  CMD17|  R1 + DAT 数据块    |  从内部 memory 返回 512B + CRC16

内部存储: logic [7:0] mem[0:TOTAL_SECTORS*512-1]，通过 $readmemh 或 task 预加载 FAT32 镜像

错误注入接口 (通过 config_db 控制):
  inject_crc_error        — 响应帧 CRC 字段污染
  inject_timeout          — 卡不响应 (验证 DUT timeout 路径)
  inject_wrong_cmd_index  — 响应帧 cmd index 字段错误 (验证 DUT syntaxe 路径)

Ncr 延迟可配置: 支持 min(2周期) 到 max(64周期) 覆盖

--------------------------------------------------------------------------------
五、FAT32 镜像生成器 (fat32_image_gen.sv)
--------------------------------------------------------------------------------

在仿真初始化时构造合法 FAT32 磁盘布局，写入 SD 卡模型 memory:
  Sector 0   : MBR (partition table offset 446，type=0x0C，start_lba 可配置)
  Boot sector: 合法 BPB
                 - sectors_per_cluster (0x0D)
                 - reserved_sectors   (0x0E)
                 - num_fats           (0x10)
                 - sectors_per_fat    (0x24)
                 - root_cluster       (0x2C)
  根目录扇区 : 包含 "IMAGE   BIN" 的 32 字节目录项 (起始簇号可配置)
  文件数据   : 76800 字节 RGB332 测试图案 (渐变色 / 棋盘格)

六、Layer 1 — sdcmd_ctrl 模块级 UVM 测试
--------------------------------------------------------------------------------

Transaction (sdcmd_txn) 随机化字段:
  rand bit [5:0]  cmd    — 约束到 {0,2,3,7,8,16,17,55,41}
  rand bit [31:0] arg
  rand bit [15:0] clkdiv — 约束到 [1:200]
  rand bit [15:0] precnt — 约束到 [46:500]
  bit expect_timeout
  bit expect_crc_error

Scoreboard 检查项:
  - 命令帧: start bit=0, trans bit=1, cmd index, arg, CRC7, end bit=1
  - 响应帧: start bit=0, trans bit=0, cmd index echo, CRC7 (R2/R3 除外), end bit=1
  - DUT 输出: done/timeout/syntaxe 与预期一致
  - resparg 值与响应帧 [39:8] 一致

覆盖率目标:
  cg_cmd    : 覆盖所有 cmd index (bins: CMD0/8/55/41/2/3/7/16/17)
  cg_result : 覆盖 done / timeout / syntaxe 三种结果
  cg_timing : clkdiv 范围 (min/mid/max) × precnt 范围
  cross      : cmd × result

测试用例:
  sdcmd_smoke_test      — CMD8 + R7 响应，验证 resparg=0x1AA
  sdcmd_init_test       — 完整初始化序列 CMD0→CMD8→…→CMD16
  sdcmd_timeout_test    — 卡不响应，验证 timeout=1
  sdcmd_crc_error_test  — 注入错误 CRC，验证 syntaxe=1
  sdcmd_r2_test         — CMD2 136-bit 长响应接收
  sdcmd_rand_test       — N 条随机命令 + 覆盖率驱动

  sd_reader_multi_read_test   — 连续读多个不同扇区
  sd_reader_init_retry_test   — ACMD41 模拟多次 busy 后就绪 (可配置循环次数)

八、Layer 3 — 全系统端到端验证
--------------------------------------------------------------------------------

FAT32 Scoreboard 监控点:
  - sd_file_reader 的 ram_addr / ram_wdata / ram_we 信号
  - 验证写入 img_ram 的 76800 字节 = SD 卡模型 IMAGE.BIN 数据
  - file_found=1 正确置位
  - read_done=1 正确置位

VGA Monitor 校验 (passive):
  - hsync/vsync 时序: H_TOTAL=800, V_TOTAL=525
  - Hsync pulse=96, Vsync pulse=2
  - H front porch=16, H back porch=48
  - V front porch=10, V back porch=33
  - 采样有效区像素，重建 320×240 帧 (考虑 2x 放大还原)
  - 逐像素比对 RGB332 原始数据

测试用例:
  sys_normal_test               — 完整流程端到端黄金比对
  sys_no_file_test              — 根目录无 IMAGE.BIN，验证 ERROR 状态
  sys_different_cluster_size    — sectors_per_cluster = {1, 8, 32, 64}
  sys_different_partition_lba   — 测试不同 MBR 分区起始 LBA
  sys_large_reserved_sectors    — 较大 reserved_sectors 值

--------------------------------------------------------------------------------
十一、关键设计约束与验证注意事项
--------------------------------------------------------------------------------

1. FAT32 连续簇假设:
     sd_file_reader 假设 IMAGE.BIN 连续存储，不跟踪 FAT 链。
     验证时保持同样假设，同时增加 negative test 验证非连续场景行为。

2. 1-bit 数据线模式:
     sd_reader 仅使用 DAT[0]，DAT[3:1] 被 top 拉高。
     SD 卡模型仅需实现 1-bit 传输。

3. CRC16 数据块校验:
     DUT 读取 512B + 16-bit CRC 但不校验 CRC16（仅读出忽略）。
     SD 卡模型需正确生成 CRC16 (多项式 0x1021) 以发现潜在误采样。

4. 时钟切换:
     CMD7 完成后 sd_reader 切换到快速时钟 (FASTCLKDIV)。
     Layer2 测试需覆盖时钟切换前后的时序正确性。

5. 仿真加速:
     sd_reader 的 SIMULATE=1 参数缩短初始化等待周期。
     Layer2/3 仿真使用 SIMULATE=1 以减少运行时间。
     Layer1 仿真不限制（sdcmd_ctrl 本身无此参数）。

6. 复位:
     所有模块 active-low rstn / rst_n。
     测试开始时给足 reset 时间 (至少 10 个 sdclk 周期)。
